온라인 스무고개 - 테스트 가이드
이 가이드는 '온라인 스무고개' 게임을 처음 테스트하는 분들을 위한 설명서입니다.

중요: 이 게임은 3명 또는 4명이 함께해야 시작할 수 있습니다.

[테스트 전 준비 사항]
컴퓨터에 .NET 9 SDK (이상)가 설치되어 있어야 합니다.

컴퓨터에 Git이 설치되어 있어야 합니다.

[1단계: 프로젝트 다운로드]
'명령 프롬프트(CMD)' 또는 'PowerShell'을 엽니다. (시작 메뉴에서 cmd 검색)

프로젝트를 저장할 폴더로 이동합니다. (예: 바탕화면으로 이동) 
cd Desktop

아래 명령어를 입력해 깃허브에서 프로젝트를 복사합니다. 
git clone https://github.com/metgugu/GameServer2025-2.git

방금 생성된 폴더로 이동합니다. 
cd GameServer2025-2

프로젝트 실행에 필요한 부품(패키지)을 설치합니다. (최초 1회) 
dotnet build

[2단계: 게임 실행]
게임을 실행하는 방법은 2가지가 있습니다. 한글로 완벽하게 테스트하려면 [방법 A]를, 영어로만 간편하게 테스트하려면 [방법 B]를 선택하세요.

[방법 A: 완벽 테스트 (한글 O)] (권장)
특징: 한글 입력이 완벽하게 작동하며, 키보드 씹힘 현상이 없습니다.

환경: CMD 또는 PowerShell

CMD 또는 PowerShell 창을 테스트 인원수만큼 엽니다. (총 3~4개)

모든 창에서 cd 명령어로 1단계에서 다운로드한 폴더로 이동합니다. 
예: cd C:\Users\YourName\Desktop\GameServer2025-2

[1번 창 (서버)] 서버 프로그램을 실행합니다. 
.\OTQ.Server\bin\Debug\net9.0\OTQ.Server.exe 
("Starting server..." 메시지가 나오면 성공입니다. 이 창은 끄지 마세요.)

[2, 3, 4번 창 (클라이언트)] 클라이언트 창에서는 반드시 2개의 명령어를 순서대로 입력해야 합니다.

첫째, 한글 모드(chcp 65001)로 변경합니다. 
chcp 65001 
(Active code page: 65001 메시지가 뜹니다)

둘째, 클라이언트를 실행합니다. 
.\OTQ.Client\bin\Debug\net9.0\OTQ.Client.exe

모든 클라이언트가 접속하면, 가장 먼저 접속한 클라이언트(방장)가 "/start"를 입력하여 게임을 시작합니다.

[방법 B: 간편 테스트 (영어 전용)]
특징: VS Code에서 바로 실행할 수 있지만, 반드시 영어로만 테스트해야 합니다.

주의: 한글(정답, 질문) 입력 시 "???"로 깨집니다.

VS Code(Visual Studio Code) 프로그램으로 1단계에서 다운로드한 'GameServer2025-2' 폴더를 엽니다.

VS Code 내장 터미널을 총 3~4개 엽니다. (상단 메뉴: 터미널 > 새 터미널)

[1번 터미널 (서버)] 
dotnet run --project OTQ.Server

[2, 3, 4번 터미널 (클라이언트)] 
dotnet run --project OTQ.Client

방장이 "/start"를 입력하여 게임을 시작합니다. (주의: 닉네임, 정답, 질문 등 모든 입력은 반드시 영어로만 하셔야 합니다.)
